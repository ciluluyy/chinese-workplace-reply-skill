# 关系与权力方向手册

## 目录

- 领导或权威方
- 善意同事或支持者
- 难相处的同事
- 客户与外部合作方
- 普通同事、下属与群聊
- 判断不清时

## 领导或权威方

目标是让对方快速判断，同时保留用户的专业边界。

### 推荐结构

1. 先给结论或已确认事实。
2. 给必要的影响或取舍，不写成长篇解释。
3. 需要领导决定时，把优先级还给领导。

### 常用动作

- 接受明确任务：`好的，我按 X 处理，预计 Y 前反馈。`
- 任务边界不清：`可以配合，想先确认一下这次需要我负责 X，还是也包含 Y？`
- 工作量冲突：`这项可以先推进，不过会影响原定的 X，您看两项按哪个优先？`
- 不同意见：`我理解您的考虑。结合目前的 X 情况，我更建议 Y，主要风险是 Z。`
- 尚未确认：`目前掌握的是 X，Y 还需要核实，我确认后再向您反馈。`

### 避免

- 只说“做不了”，不给事实或取舍。
- 为显积极主动，把统筹、协调、驻点或长期支持全部接下。
- 用户只接受其中一项职责时，不要为了显得配合，再主动认领争议职责中的“汇总、对接、临时帮忙”等子任务。
- 不要把所有拒绝项逐条念给领导；能用“全市统筹事项”“其他县市区相关职责”等中性范围概括时，优先概括。
- 把建议写成命令，或把不同意见写成自我辩护。

## 善意同事或支持者

目标是接住善意，留下合作感，但不写成正式感谢信。

### 推荐结构

`认可/感谢 + 轻量回应 + 自然收尾`

示例动作：

- `好嘞，感谢支持，后面咱们及时沟通。`
- `好的，您这个安排很周到，后续有情况我们再对一下。`
- `明白，感谢您从实际工作角度考虑。[握手]`

### 避免

- 连续堆“感谢、辛苦、麻烦、支持”。
- 因为对方友善就追加“有事都找我”“我都可以配合”。

## 难相处的同事

目标不是赢一场嘴仗，而是修正框架、留下记录、结束纠缠。

### 四种情况

1. **甩锅：** 只确认职责和事实，不评价人。
   - `这部分之前确定由 X 负责，我这边负责的是 Y，咱们按原分工推进就行。`
2. **暗讽：** 轻轻接住字面意思，再回到事情。
   - `建议挺有想象力，不过咱们还是先按已经确认的方案落地吧。`
3. **越界安排：** 说明可配合范围和决定人。
   - `这部分我可以提供信息，具体安排还需要由 X 确认。`
4. **反复纠缠：** 重复结论，不新增解释。
   - `前面的结论不变，后续按 X 执行即可。`

### 避免

- 解释过多，让对方获得新的争论材料。
- “你是不是”“你总是”等人格归因。
- 群内公开讽刺；需纠偏时写事实，需谈感受时私聊。

## 客户与外部合作方

目标是让对方感到事情被接住，同时不虚构进度、责任或承诺。

### 推荐结构

`确认影响/诉求 + 已核实状态 + 明确下一步或下次更新时间`

示例动作：

- `收到，确实会影响您这边的操作体验。我们正在核实原因，预计今天 17:00 前先同步一次进展。`
- `理解您希望尽快确认。目前 X 已完成，Y 还在处理，我在〔时间〕前给您明确结果。`
- `这个需求我们先记录下来，当前版本暂不支持；我会把业务场景一并反馈给产品评估。`

### 避免

- 未核实便道歉并认责。
- 用“尽快”代替时间点，又不给下次更新时间。
- 为安抚对方承诺无法控制的结果。

## 普通同事、下属与群聊

### 普通同事

平等、直接，说明动作和时间。不要使用管理层话术包装简单协作。

### 下属

任务要清晰：背景、交付物、截止时间、判断权限。指出问题时写行为和影响，不贴标签。

### 群聊

- 只回复公共事实、决定和下一步。
- 涉及责任争议、绩效、情绪或个人失误时转私聊。
- 群内纠偏可用：`补充说明一下，当前确认的口径是 X，后续统一按这个执行。`

## 判断不清时

从称呼、语气和动作词判断：

- `请你/安排一下/统筹`：可能是上级或项目权威。
- `麻烦配合/能否支持`：可能是跨部门或外部合作方，权力方向仍需结合称呼。
- `您/老师/主任/总`：通常要求更稳妥，但不自动代表对方是领导。
- 群聊截图：先识别最后一条需要回应的消息，再确认用户是否代表个人或单位发言。

若两种身份会导致相反回复，问：`对方是您的领导，还是平级同事？` 其他情况下直接给稳妥版本。
